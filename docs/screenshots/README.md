# Screenshots

Drop real device captures here to replace the rendered mockups in the main README
(`docs/preview.svg`).

## Suggested set

| File | Screen |
|------|--------|
| `login.png` | Phone-number login |
| `discover.png` | Map discovery (branch markers + card carousel) |
| `products.png` | Catalogue (category tabs + product grid) |
| `basket.png` | Basket & checkout (Card / Cash) |
| `scanner.png` | Staff QR scanner / product editor |
| `orders.png` | Staff orders queue |

## How to capture

- **iOS Simulator** — `⌘S` (saves to Desktop), or _Device → Screenshot_.
- **Android emulator** — the camera button in the toolbar, or
  `adb exec-out screencap -p > products.png`.
- A short **demo GIF** (e.g. [Kap](https://getkap.co) or `licecap`) embedded at the
  top of the README is even more compelling than stills.

Once the real captures are in, point the README hero at them (a grid of `<img>` in a
`<p align="center">` block) instead of `docs/preview.svg`.
